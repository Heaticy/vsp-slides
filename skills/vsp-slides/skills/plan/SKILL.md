# VSP Slides Plan

用于生成前规划，不直接改写或渲染 PPT，除非用户明确要求继续执行。

## 输入

- 源材料路径或内容：Markdown、PDF、讲稿、论文、课程材料等
- 演讲场景：论文汇报、课程展示、项目展示、组会、其他
- 演讲人姓名、邮箱、标题、主题偏好
- 交付目标：只要方案、生成 Markdown、HTML/PDF、是否需要审计

## 流程

1. 检查材料路径和可读性；若缺失关键路径，先询问。
2. 读取材料目录、标题层级、图片数量、公式/代码密度。
3. 按场景选择候选主题：
   - 论文/项目：`report`
   - 教学/习题：`tutorial-red`、`tutorial-red-shtu`、`tutorial-purple`
   - 轻量展示：`tutorial-nailong`
4. 规划页面结构：封面、双目录、章节过渡、正文页、总结、尾页。
5. 规划图片策略：每张重要图至少说明页 + 展示页；记录可能的路径风险。
6. 输出执行计划，包含后续建议子 skill：
   - 资源风险多：先 `assets`
   - 计划已确认：进入 `start`
   - 只生成 Markdown：进入 `generate`

## 输出

输出一份简洁计划：

- 输入材料摘要
- 推荐主题和理由
- 页面结构和预计页数
- 图片/公式/代码策略
- 产物路径建议
- 验证方法：至少包含 `render` 和 `audit`

不要在用户只要求规划时擅自生成完整 PPT。
